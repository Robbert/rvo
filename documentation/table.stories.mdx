<Description># Table</Description>

Met tabellen toon en organiseer je data in rijen en kolommen.

## Wanneer gebruik je een tabel?

Je gebruikt een tabel wanneer er data getoond wordt die vergeleken met elkaar moet kunnen worden.

## Anatomie

Een tabel bestaat uit:

1. Caption: deze beschrijft de inhoud van de tabel.
2. Table header: deze bevat de titel van de kolom.
3. Table row: deze bevat de data.

## Content richtlijnen

- Schrijf informatieve, beknopte table headers die duidelijk omschrijven welke data de kolom bevat.
- Toon meeteenheden (zoals â‚¬ or m2) op in de table header zodat de niet telkens herhaald hoeven te worden in de kolomen.
- Zorg ervoor de table headers starten met een hoofdletter en niet eindigen met een komma, puntkomma of een punt.
- Zorg ervoor dat de data in de table row begint met een hoofdletter en geen meeteenheid bevat.
- Maak consistent gebruik van decimalen.

## Best practices

### Doen

Tabellen moeten:

- Een collectie tonen van gestructureerde data.
- Een logisch kolom header hebben.
- Gebruik een caption om de tabel te beschrijven.
- Kolommen tonen op volgorde van belangrijkheid waarbij de belangrijkste kolom als eerst getoond wordt.
- Ervoor zorgen dat de context altijd bewaard blijft door gebruik te maken van headers die vast geplakt blijven.
- Spreek menselijke taal: laat alleen relevante kolommen zien die waarde hebben. Dus geen nietszeggende referentienummers.
- Maak de tabel horizontaal scrolbaar wanneer er onvoldoende ruimte is op het scherm om de tabel volledig te tonen. (bron: https://inclusive-components.design/data-tables/)
- Lijn data in tabelcellen aan de bovenkant uit om de scanbaarheid te vergroten. (bron: https://www.uxmatters.com/mt/archives/2020/07/designing-mobile-tables.php)
-

### Niet doen

- Verander de layout niet wanneer tabellen op een kleiner scherm getoond moet worden. Hierdoor verdwijnt de onderliggende semantiek van de tabel. (bron: https://inclusive-components.design/data-tables/)
